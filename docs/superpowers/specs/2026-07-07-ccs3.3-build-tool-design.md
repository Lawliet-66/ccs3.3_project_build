# CCS3.3 Project Build Tool — 设计文档

## 概述

使用 Python 封装 TI CCS3.3 的 `timake` 命令行工具，提供统一的 CLI 接口来构建和清理 DSP 项目。打包为独立 exe，无需用户安装 Python 环境。

## 命令行接口

```
ccs3.3_project_build.exe <project.pjt> [-clean] -log <log_dir>
```

| 参数 | 说明 |
|---|---|
| `project.pjt` | 项目文件路径（支持 `*.pjt` glob 匹配，取第一个） |
| `-clean` | 可选，执行 clean 构建 |
| `-log <log_dir>` | **必需**，日志输出目录（不存在时自动创建） |

**示例：**
```
ccs3.3_project_build.exe *.pjt -clean -log ./log_files
ccs3.3_project_build.exe KADD008U2.PJT -log ./log_files
ccs3.3_project_build.exe ./APP/KADD008U2.PJT -log ./log_files
```

## 核心执行流程

1. **参数解析** — 解析 `.pjt`（glob 取第一个）、`-clean`、`-log` 目录
2. **构建 timake 命令**：
   - 带 `-clean`：`timake <project.pjt> Debug -clean`
   - 不带 `-clean`：`timake <project.pjt> Debug`
3. **执行 timake** — 通过 `subprocess.Popen` 启动，实时捕获 stdout + stderr
4. **写入日志** — 完整输出写入 `{log_dir}/{Clean|Debug}_{YYYYMMDD_HHMMSS}.log`
5. **检测构建结果**：
   - **Clean**：输出包含 `"Build Clean Complete"` → success，否则 fail
   - **Build**：输出包含 `"Build Complete"` **且** `"0 Errors"` → success，否则 fail
6. **输出 JSON** 到 stdout：`{"status": "success|fail", "log": "path/to/log_file.log"}`
7. **退出码**：success → 0，fail → 1

## 状态检测规则

| 操作 | 成功标志 | 失败场景 |
|---|---|---|
| `-clean` | stdout 包含 "Build Clean Complete" | 未包含该字符串 |
| 构建（无 `-clean`） | stdout 包含 "Build Complete" **且** 正则匹配 `(?<!\d)0 Errors`（独立数字 0，避免 "10/100 Errors" 误匹配） | "N Errors"（N≠0）、未包含 "Build Complete" 或 timake 进程退出码非0 |

## 异常处理

- **timake 未找到**：捕获 `FileNotFoundError`，输出 `{"status": "fail", "log": ""}`，退出码 1
- **timake 进程异常退出**：非0退出码均视为 fail
- **日志目录不可写**：捕获 `PermissionError` / `OSError`，输出 JSON 到 stderr，退出码 1
- **`.pjt` 文件未找到**：glob 无匹配或指定文件不存在，输出 JSON 到 stderr，退出码 1

## 技术选型

- **语言**：Python 3.13（仅使用标准库）
- **打包**：PyInstaller 6.x+（`--onefile` 模式）
- **外部依赖**：timake（位于 `G:\CCStudio_v3.3PLA\cc\bin\timake`）

## 测试方案

| 测试场景 | 操作 | 预期结果 |
|---|---|---|
| 正常清理 | `ccs3.3_project_build.exe *.pjt -clean -log ./log_files` | `{"status": "success"}`，日志含 "Build Clean Complete" |
| 正常构建 | `ccs3.3_project_build.exe *.pjt -log ./log_files` | `{"status": "success"}`，日志含 "Build Complete" 和 "0 Errors" |
| 构建失败 | 故意修改 `Compile_Information.c` 引入语法错误后执行构建 | `{"status": "fail"}` |
| 项目文件不存在 | `ccs3.3_project_build.exe nonexist.pjt -log ./log_files` | `{"status": "fail"}`，错误信息明确 |
